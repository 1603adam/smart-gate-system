02 · Overview

Introduction

The rapid evolution of smart home technology has moved residential security away from the vulnerabilities of physical keys and remote controls. This project presents the Smart RFID & Keypad Automated Gate System with Night-Light Integration.

By merging hardware automation with multi-factor authentication, this prototype bridges robust perimeter security with everyday user convenience.

🔐 RFID Auth 🔢 PIN Keypad 🌙 Night Light 📡 Ultrasonic 📺 LCD Display
System Overview
03 · Problem

Problem Statement

🔑
Single Point of Failure

Relying on physical keys or remote fobs — if lost or forgotten, residents are locked out and properties are exposed.

👥
No Guest Access Method

Legacy systems offer no secure, independent access methods for visitors, couriers, or emergency services.

💡
Inefficient Lighting

Conventional security lighting uses simple timers or stays manual — causing dark driveways at night or wasted energy during daytime.

✅
Our Solution

Dual-factor authentication (RFID + PIN) paired with LDR-based energy-saving automatic driveway lighting.

05 · Logic Gates

Circuit Diagram

The Boolean equations derived from the K-Map are implemented using AND gates. Two independent AND gates govern each output independently.

GATE = A · B
LIGHT = A · C

Input A (CAR) branches to both gates.
Input B (AUTH) pairs with A → GATE output
Input C (LDR) pairs with A → LIGHT output
The C variable drops from the GATE equation; B drops from the LIGHT equation.

06 · Hardware

IoT Components

🧠
Arduino Uno
Central processing unit — executes control logic for gate & lighting subsystems
📡
MFRC522 RFID
Primary resident authentication via contactless RFID card scanning
📏
HC-SR04 Ultrasonic
Measures vehicle proximity to trigger "Car Present" logic
🔢
4×4 Keypad
Secondary auth — 4-digit PIN access for guests & family members
☀️
LDR Photoresistor
Monitors ambient light — determines Night vs Day for auto lighting
⚙️
Servo Motor
Gate actuator — opens when authorized access is validated
🖥️
LCD 1602 (I2C)
Displays real-time status updates & visual feedback to user
🔴🟢⚪
LED Indicators
Red = Denied · Green = Granted · White = Driveway

08 · Programming

IoT Coding

#include <SPI.h>
#include <MFRC522.h>
#include <Servo.h>
#include <Keypad.h>

// Core logic — GATE = AB, LIGHT = AC
bool A = getDistance() < 50; // Car
bool B = isAuthorized(); // Auth
bool C = isDark(); // LDR

if (A && B) openGate();
if (A && C) lightOn();

Libraries: SPI · MFRC522 · Servo · Wire · LiquidCrystal_I2C · Keypad
Key functions: getDistance() · isOwnerCard() · handleAccessGranted() · handleAccessDenied()
